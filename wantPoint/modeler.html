QGIS 모델러 실습

1. 공간처리 프레임워크 개요

  • 공간처리 프레임워크란

    • QGIS에서 내장 및 제3자 알고리즘을 호출하는 데 사용되는 지리 공간처리 환경

    • 핵심 플러그인으로서 기본으로 설치되지안 사용자가 활성화시켜야 함

      • 플러그인 → 플러그인 관리 및 설치 → 설치됨 탭 → Processing 항목 체크

    • 프레임워크 GUI에는 4개의 기본 항목들이 있음

      • 툴박스(ToolBox)
      • 그래픽 모델러(Graphical Modeler)
      • 이력 관리자(History)
      • 배치 프로세스(Batch Processing, 공간 처리 탭에는 해당 인터페이스가 없음)

  • 툴박스

    • GUI의 메인 항목으로 단일 알고리즘을 실행하거나 해당 알고리즘에 기반한 배치 프로세스(batch process) 실행
      • 메뉴 바에서 공간처리 → 툴박스 또는 보기 → 패널 → 공간 처리 툴박스

  • 그래픽 모델러

    • 모델 생성기의 그래픽 인터페이스를 통해 여러 알고리즘을 결합시켜, 여러 하위 프로세스를 이용하는 단일 프로세스를 생성하는 작업 흐름 정의

  • 이력 관리자

    • 이력 파일에 앞서서 언급한 항목을 사용해서 수행된 모든 액션을 저장하고, 추후 이력 관리자를 통해 쉽게 재수행 가능

  • 배치 프로세싱

    • 여러 데이터셋에 대해 단일 알고리즘을 수행하는 과정을 자동화할 수 있음

    • 각 툴 하단 배치 프로세스로 실행 버튼을 통해 인터페이스에 접근

  • 2. 그래픽 모델 생성기

  • 그래픽 모델 생성의 기본 이해

    • 사용하기 쉽고 간단한 인터페이스를 통해 복잡한 모델을 생성할 수 있게 함

    • GIS 작업 시 연속적인 공간 처리 작업들을 단일 공간처리 작업으로 묶을 수 있음

    • 서로 다른 입력 집합에 대해 여러 작업을 연속해서 처리하는 것보다 단일 공간 처리 작업을 실행하는 편이 더 편리

    • 몇 단계를 거치든, 몇 개의 서로 다른 알고리즘을 사용하든 상관없이 모델은 단일 알고리즘처럼 실행되기 때문에 시간과 노력 절약

  • 그래픽 모델 생성기 인터페이스

    • 모델 생성기는 모델의 구조 및 모델이 나타내는 작업 흐름을 표시하는 작업 캔버스를 보유

      • 모델 속성: 모델 및 모델을 담게 될 그룹의 이름 지정

      • 입력: 사용자 모델을 형성할 모든 입력물

      • 알고리즘: 사용할 수 있는 공간 처리 알고리즘

      • 변수: 공간 처리 모델 생성기에서만 사용할 수 있는 변수 정의

      • 실행취소 이력: 모델 생성기에서 일어나는 모든 기록이 등록되며 사용자가 실수한 내용을 쉽게 취소 가능

  • 그래픽 모델 생성 2단계

    • 필요한 입력물 정의하기

      • 입력해야 할 항목들을 Inputs 패널의 Parameters 창에 추가하여 모델 실행 시 사용자가 그 값들을 설정할 수 있음

      • 모델은 그 자체로 알고리즘으로서 공간 처리 프레임워크에서 사용할 수 있는 모든 알고리즘과 마찬가지로 파라미터 창을 자동적으로 생성

    • 작업 흐름 정의하기

      • 모델의 입력 데이터를 사용해서 알고리즘을 추가하고 해당 알고리즘이 정의된 입력물을 또는 모델 안에 있는 다른 알고리즘이 생성하는 산출물을 어떻게 사용할지 선택함으로써 작업 흐름을 정의

  • 3. 그래픽 모델러 실습

  • 작업 개요

    • 실습 목표

      • 해상 해적행위 사건의 포인트 레이어 자료를 이용하여 전세계 육각 그리드에 걸쳐 발생건수를 집계하고 그 결과를 밀도 지도로 제작하는 처리 모델을 만들고자 함

    • 실습 자료

      • 미국 국립지질정보국의 '해양안전정보 포털(https://msi.nga.mil/NGAPortal/MSI.portal)'에서 제공하는 모든 해양 해적 행위에 대한 점 자료(All ASAM Events.shp)
      • '자연 지구(http://naturalearthdata.com)'에서 제공하는 전 세계 대륙 지도(ne_10m_land.shp)

      실습자료 다운로드

  • 그래픽 모델러 실습

    1. 공간 처리 메뉴 > 그래픽 모델러를 클릭하여 실행

    2. 모델 속성에 다음의 파라미터 값을 입력하고 저장 버튼을 클릭하여 piracy_hexbin으로 저장

      • 이름: 해적 행위 밀도 분석

      • 그룹: 모델러 연습


    3. 왼쪽에서 Inputs 패널에서 "벡터 레이어"를 오른쪽 캔버스로 드래그&드롭하면 팝업되는 창에 다음과 같이 파라미터를 입력

      • 유형: 입력 포인트, 도형 설정: 포인트, 필수 체크

      • 유형: 베이스 레이어, 도형 설정: 폴리곤, 필수 체크


    4. Inputs 패널에서 "숫자 입력"을 선택하고 캔버스로 드래그&드롭한 후 다음과 같이 파라미터를 입력(모델을 변경하지 않고 사용자가 다양한 그리드 크기를 빠르게 생성 가능)

      • 설명: Grid Size

      • 번호 유형: 정수(integer)

      • 기본값: 0


    5. 입력 레이어의 좌표계를 재투영하기 위해 Toolbox에서 "레이어 재투영"을 검색하여 드래그&드롭하고 다음과 같이 파라미터를 입력

      • 입력 레이어: 모델 입력으로 변경 후 기본 레이어 선택

      • 대상 좌표계 부분에서 프로젝트 좌표계 사용 체크


    6. 그리드를 생성하기 위해 Toolbox에서 "벡터 생성 > 그리드 생성"을 검색하여 드래그&드롭하고 다음과 같이 파라미터를 입력

      • 그리드 유형: 헥사곤 폴리곤

      • 그리드 범위: 알고리즘 산출물

      • 수평, 수직간격: 모델 입력으로 변경 후 Grid Size 선택


    7. 사건 발생 그리드 위치만 추출하기 위해 Toolbox에서 "위치로추출"을 검색하여 드래그&드롭하고 다음과 같이 파라미터를 입력

      • 다음에서 피처 추출: 알고리즘 산출물로 변경 후 그리드 선택

      • 피처 위치(지오메트릭 서술자): 우측 버튼을 클릭하여 교차(intersect)로 체크

      • 다음과 같은 피처를 비교: 모델 입력으로 변경 후 입력 포인트 선택


    8. 육각 그리드 내 포인트 수를 계산하기 위해 Toolbox에서 "폴리곤에 포함하는 포인트 개수 계산"을 검색하여 드래드&드롭하고 다음과 같이 파라미터를 입력

      • 폴리곤: 알고리즘 산출물로 변경 후 "위치로 추출" 알고리즘에서의 "Extracted(위치)" 선택

      • 포인트: 모델 입력으로 변경 후 입력 포인트 선택

      • 개수: 합계 입력


    9. 저장 버튼을 클릭하여 완성된 모델을 최종적으로 저장


    10. 결과물 확인을 위해 프로젝트 좌표계를 EPSG:54009로 변경하고 툴 박스에서 모델 > 모델러 연습 > 해적 행위 밀도 분석을 찾아 실행

      • EPSG:54009는 미터 단위를 가지는 등면적 투영으로 생성되는 grid size를 100,000m(100km)로 설정
